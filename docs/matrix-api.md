# Matrix API Integration Guide

This document describes how to interact with the Matrix bot via the Matrix Client API.

## Quick Start: Test Script

For quick testing, use the built-in test script:

```bash
# Copy the example environment file
cp .env.matrix.example .env.matrix

# Edit with your configuration
# (or set via environment variables)

# Source and use
source .env.matrix
./scripts/matrix-test.sh send '!m -s'  # Send model status command
./scripts/matrix-test.sh status 5      # Show last 5 messages
./scripts/matrix-test.sh wait          # Wait for bot to be idle
```

See [scripts/matrix-test.sh](../scripts/matrix-test.sh) for full documentation.

## Prerequisites

- Homeserver URL (e.g., `http://your-homeserver:8008`)
- Access token for a user in the allowed users list
- Room ID of the bot room (e.g., `!roomid:example.com`)

## Getting an Access Token

Access tokens are generated by your Matrix homeserver. Common methods:

### Using Synapse Admin API

```bash
# Generate a new access token for an existing user
curl -X POST "http://localhost:8008/_synapse/admin/v1/users/@user:example.com/login" \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{"generate_new_access_token": true}'
```

### Using the Matrix Client API

```bash
# Login with username/password
curl -X POST "http://your-homeserver:8008/_matrix/client/r0/login" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "m.login.password",
    "user": "username",
    "password": "password"
  }'
```

The response contains an `access_token` field.

## Sending a Message to the Bot

```bash
#!/bin/bash

# Configuration
HOMESERVER="http://your-homeserver:8008"
ACCESS_TOKEN="your-access-token-here"
ROOM_ID="!roomid:example.com"

# Send a text message
MESSAGE='{"msgtype": "m.text", "body": "Hello, bot!"}'

curl -X POST "$HOMESERVER/_matrix/client/r0/rooms/$ROOM_ID/send/m.room.message" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "$MESSAGE"
```

## Getting Room Members

```bash
curl -s "$HOMESERVER/_matrix/client/r0/rooms/$ROOM_ID/members" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | \
  python3 -c "import sys, json; data = json.load(sys.stdin); print('\n'.join([m['user_id'] for m in data.get('chunk', [])]))"
```

## Verifying Message Delivery

```bash
# Check recent events in the room
curl -s "$HOMESERVER/_matrix/client/r0/rooms/$ROOM_ID/messages?dir=b&limit=10" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | \
  python3 -c "import sys, json; data = json.load(sys.stdin); print(json.dumps(data, indent=2))"
```

## Using Python

```python
import requests
import json

# Configuration
HOMESERVER = "http://your-homeserver:8008"
ACCESS_TOKEN = "your-access-token-here"
ROOM_ID = "!roomid:example.com"

headers = {
    "Authorization": f"Bearer {ACCESS_TOKEN}",
    "Content-Type": "application/json"
}

# Send a message
message = {
    "msgtype": "m.text",
    "body": "Hello, bot!"
}

response = requests.post(
    f"{HOMESERVER}/_matrix/client/r0/rooms/{ROOM_ID}/send/m.room.message",
    headers=headers,
    json=message
)

print(f"Status: {response.status_code}")
print(f"Response: {response.json()}")
```

## Using Node.js

```javascript
const axios = require('axios');

const HOMESERVER = 'http://your-homeserver:8008';
const ACCESS_TOKEN = 'your-access-token-here';
const ROOM_ID = '!roomid:example.com';

async function sendMessage(body) {
  const response = await axios.post(
    `${HOMESERVER}/_matrix/client/r0/rooms/${ROOM_ID}/send/m.room.message`,
    {
      msgtype: 'm.text',
      body: body
    },
    {
      headers: {
        'Authorization': `Bearer ${ACCESS_TOKEN}`,
        'Content-Type': 'application/json'
      }
    }
  );
  return response.data;
}

sendMessage('Hello, bot!').then(console.log);
```

## Environment Variables for Testing

Copy `.env.matrix.example` to `.env.matrix` and configure:

```bash
cp .env.matrix.example .env.matrix
```

Then edit `.env.matrix`:

```bash
# .env.matrix
MATRIX_HOMESERVER="http://your-homeserver:8008"
MATRIX_ACCESS_TOKEN="your-access-token"
MATRIX_ROOM_ID="!roomid:example.com"
MATRIX_ADMIN_ACCESS_TOKEN="your-admin-token"  # Optional, for @pi-admin
```

Then source it in your scripts:

```bash
source .env.matrix
curl -X POST "$MATRIX_HOMESERVER/_matrix/client/r0/rooms/$MATRIX_ROOM_ID/send/m.room.message" \
  -H "Authorization: Bearer $MATRIX_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"msgtype":"m.text","body":"test"}'
```

**Tip:** Use the test script for easier interaction:

```bash
source .env.matrix
./scripts/matrix-test.sh send '!m -s'  # Send command
./scripts/matrix-test.sh status 10     # Show last 10 messages
```

## Notes

- The bot only responds to users in the `allowedUserIds` list in its configuration
- The bot only responds in rooms listed in `allowedRoomIds`
- Messages must be sent with `msgtype: "m.text"` for text messages
- The bot's response will appear as a new message in the room

## Troubleshooting

### "M_FORBIDDEN" Error

This means the access token is invalid or doesn't have permission to access the room.

### "M_NOT_FOUND" Error

The room ID is incorrect or the user is not a member of the room.

### No Response from Bot

- Verify the user is in the `allowedUserIds` list
- Verify the room is in the `allowedRoomIds` list
- Check the bot's logs for errors
